## Target User & Narrowest Wedge

**Primary:** Beltway security auditor (173.66.27.4 profile) — runs package vulnerability scans as part of vendor trust checks. Next natural call after package scan: "who is this vendor registered with?" (company registry) and "who owns the IP block serving this API?" (ASN).

**Secondary:** Chinese IP researcher — patent citations + domain history already in use. Company registry is the missing piece for full IP due diligence workflow.

**Narrowest wedge for T02:** Three tools.
- `lookup_global_company_registry(name_or_id, countries[])` — GLEIF LEI API (free, no key, 2M+ legal entities globally including China). Returns: legal name, jurisdiction, registration status, LEI code, registered address. Cache 24h.
- `fetch_company_officers(company_id, jurisdiction)` — Companies House UK API (free, no key). Returns: officers, roles, appointment dates. Cache 24h. UK-only for now; ASIC AU can be added in Sprint 7.
- `resolve_company_lei(company_name)` — GLEIF fuzzy name search → LEI code. Thin wrapper over GLEIF search endpoint. Cache 24h.

**Narrowest wedge for ASN:** Two tools.
- `fetch_asn_info(asn_or_ip)` — Team Cymru Bulk Whois API (free, covers all RIRs globally including APNIC/RIPE — correct choice given CN + US user segments). Input: IP address or ASN number. Returns: ASN, organization, country, RIR, description. Cache 4h.
- `fetch_ip_prefix_ownership(cidr)` — RIPE NCC REST API (free, no key). Input: IP prefix (e.g., "8.8.8.0/24"). Returns: owning organization, allocated range, country, abuse contact. Complements `fetch_asn_info` which returns the AS-level view; this returns the prefix-level view. Cache 4h.
